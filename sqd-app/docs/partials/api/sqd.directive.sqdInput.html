sqdInput
directive in module sqd

Description

Creates an input/textarea with some UI elements to allow the user to e.g. delete all text and also see if input is valid. Also allows the input to be scanned for attachable content from the web

Usage

as element:
<sqd-input
       type="{string}"
       placeholder="{string}"
       name="{string}"
       required
       maxlength="{int}"
       model="{string or object}"
       ng-readonly="{string}"
       valid="{bool}"
       textarea="{bool}"
       form="{object}"
       colour="{string}"
       background="{string}"
       links="{bool}"
       nospaces="{bool}">
</sqd-input>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
typestring

Same a for html input element

placeholderstring

Same a for html input element

namestring

Same a for html input element

requiredstring

Same as for html input element

maxlengthint

Make the input/textarea invalid if its length is greater than this

modelstring or object

Holds the data associated with the input, note this can be just text (use string) and/or media (use object) depending on whether checkForLinks has been selected as an attribute. If set as an object it will have the form {text:[],media:[]}

ng-readonlystring

Same a for html input element with angularJS

validbool

Is the input valid or not

textareabool

Is the input a textarea

formobject

Form element that this input belongs to

colourstring

Class name for the colour of the input

backgroundstring

Class name for the background colour of the view that the input is inside of

linksbool

If true the content of the input is checked to see if it contains links, the content of which is then attached to the media key of the model object. Note "model" must be an object not a string for this

nospacesbool

If true then the user will be forbidden from typing a space into the input
