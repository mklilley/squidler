stores
service in module sqd

Description

This is a service that is returned as part of storesProvider. Used to interface with the html5 local storage

Usage

stores($q, $localStorage, exceptions);

Parameters

ParamTypeDetails
$qobject

Angular promise service

$localStorageobject

Third party service to access html5 local storage

exceptionsobject

Service to catch a failed promise

Returns

object

Service object exposing methods - create, read, update

Methods

  • create(name, id, data)

    Saves some data in the local storage, but only if it doesnt exist already. It if does exist nothing is changed, update should be used instead.

    Parameters
    ParamTypeDetails
    namestring

    Name of the type of entry you want to save e.g. 'squidles' (Note, this should be plural)

    id
    (optional)
    		string

    Unique id of the entry to be saved. If not specified then an empty object will be created under "name" if it did not already exist, otherwise nothing is changed.

    dataobject

    Data object to be saved

    Returns
    promise

    Resolves to a string of the form - name + " with id= " + id + " successfully saved to local storage" if the data is stored successfully, otherwise the promise is rejected

    Example

    stores.create('squidles','VJ08tg6', {...squidle data...})

  • read(name, id)

    Reads some data in the local storage

    Parameters
    ParamTypeDetails
    namestring

    Name of the type of entry you want to read e.g 'squidles' (Note, this should be plural)

    id
    (optional)
    		string

    Unique id of the entry to be read. If not specified all entries under "name" will be returned

    Returns
    promise

    Resolves to an object specific to the type of data stored if the data is stored successfully, otherwise the promise is rejected

    Example

    stores.read('squidles','VJ08tg6')

  • remove(name, id, fields)

    Removes some data in the local storage

    Parameters
    ParamTypeDetails
    namestring

    Name of the type of entry you want to remove e.g. 'tempSquidle'

    id
    (optional)
    		string

    Unique id of the entry to be removed (if not present then all ids will be removed and the store will be reset to {})

    fields
    (optional)
    		array

    Array of strings that are the keys to be delted. NOTE if not present then the entire data entry will be delted from the local storage

    Returns
    promise

    Resolves to a string of the form - 'Entries from ' name " with id= " id " successfully deleted from the local storage" if the data is was deleted successfully, otherwise the promise is rejected

    Example

    stores.remove('tempSquidle','prize',['image'])

  • update(name, id, data)

    Updated some data in the local storage

    Parameters
    ParamTypeDetails
    namestring

    Name of the type of entry you want to update e.g. 'squidles' (Note, this should be plural)

    idstring

    Unique id of the entry to be updated

    dataobject

    Object with updated data, note not all original data keys need to be present, only ones that you want to change

    Returns
    promise

    Resolves to a string of the form - name + " with id= " + id + " successfully updated" if the data is updated successfully, otherwise the promise is rejected

    Example

    stores.update('squidles','VJ08tg6', {...squidle data...})
