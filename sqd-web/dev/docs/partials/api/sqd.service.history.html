history
service in module sqd

Description

This is a service that is returned as part of historyProvider. Used to retrieve a history of the Squidles that have either been created or viewed by the user

Usage

history($q, resources, squidles, stores, exceptions);

Parameters

ParamTypeDetails
$qobject

Angular promise service

resourcesobject

Service to communicate with backend resources

squidlesobject

Service to handle creating/reading/updating Squidles

storesobject

Service used to interface with the html5 local storage

exceptionsobject

Service to catch a failed promise

Returns

object

Service object exposing methods - read

Methods

  • read(options)

    Retrieves the Squidle history of a user

    Parameters
    ParamTypeDetails
    options
    (optional)
    		object

    If it contains the key "refresh" with value true then each entry of the history will be refreshed, otherwise only new squidles will be added

    Returns
    promise

    Resolves to an array of squidle objects (see squidles.read for more details on the structure of those objects)

    Example

    history.read()
