profiles
service in module sqd

Description

This is a service that is returned as part of profilesProvider. Used to retrieve and update a user's profile

Usage

profiles($q, stores, resources, files, exceptions);

Parameters

ParamTypeDetails
$qobject

Angular promise service

storesobject

Service used to interface with the html5 local storage

resourcesobject

Service to communicate with backend resources

filesobject

Service to upload images to the backend

exceptionsobject

Service to catch a failed promise

Returns

object

Service object exposing methods - read, update

Methods

  • read(username, options)

    Retrives the public profile for a specific user or all profiles stored locally.

    Parameters
    ParamTypeDetails
    username
    (optional)
    		string

    Username of user. If not present then all locally stored profiles are returned

    options
    (optional)
    		object

    If presented with the key "refresh" set to true then the requested profile/s will be locally updated from the server data

    Returns
    promise

    Resolves to an object with keys: avatar, location, bio and name if profile was successfully read, otherwise the promise is rejected.

    Example

    profiles.read('joblogs')

  • update(username, profile)

    Updates the public profile of a speciic user (need to be the user in question to do this)

    Parameters
    ParamTypeDetails
    usernamestring

    Username

    profileobject

    Data entries that the user wants to update, e.g. bio, avatar, location, etc...

    Returns
    promise

    Resolves to an object with keys: avatar, location, bio and name if profile was successfully read, otherwise the promise is rejected.

    Example

    profiles.read('joblogs',{bio:'Me me Me', location:'London'})
